# プロジェクト運用・デザインガイドライン

**最終更新日:** 2026年2月15日

本ドキュメントは、ウェブサイト「安塚町自治会 新田南2組 デジタル回覧板」の運用ルール、デザイン規定、および更新手順をまとめたものです。

---

## 1. 回覧板 (Kairanban)

### アーカイブページ (`/kairanban/archive`)
- **目的**: 過去の回覧板（バックナンバー）を一覧表示する。
- **デザイン規定**:
  - **テーブル形式**: 「第X号」「発行日」「回覧物（タイトル）」の3列構成。
  - **発行日**: 中央揃え (`text-center`) とする。
  - **タイトル**: 番号プレフィックス（"1. "など）は削除して表示する。
  - **リンク**: 「第X号」のテキストに個別ページへのリンクを設定する。
  - **フッター**: 不要な説明カードは削除し、以下のプライバシー注釈のみを表示する。
    > ※個人情報保護のため、公開していない資料もあります。詳細は組長へお問い合わせください。

### トップページ (`/kairanban`)
- **連絡方法の案内**:
  - ページ下部のフォームではなく、**ページ上部の「確認済みの連絡をする」ボタン**へ誘導する文言を使用する。
  - **文言例**: 「...ページ上部の『確認済みの連絡をする』ボタンよりご連絡をお願いします。」

## 2. 活動報告 (Activity Reports)

### 構成
- **一覧ページ**: `app/reports/page.tsx`
- **詳細ページデータ**: `app/reports/[id]/page.tsx` ( `reportsData` オブジェクト内)
- **画像フォルダ**: `public/reports/[日付]/`

### 運用ルール
1. **写真がない場合**:
   - サムネイルには `public/shrine.jpg` (神社の写真) を使用する（`imageUrl: "/shrine.jpg"`）。
   - `content` 内の画像表示コードは空にするか削除する。
2. **タイトル表記**:
   - 読み仮名（フリガナ）は含めない。
   - **NG**: `御鍬祭（おくわさい）の開催`
   - **OK**: `御鍬祭の開催`
3. **新規追加手順**:
   - `app/reports/[id]/page.tsx` の `reportsData` に新しいID（日付など）のエントリーを追加する。
   - `content` はMarkdown形式で記述する。
   - `generateStaticParams` に新しいIDを追加する。
   - `app/reports/page.tsx` の `reports` 配列の先頭に概要データを追加する。

## 3. デプロイ手順 (Deployment)

GitHub Pagesへの反映手順です。

1.  **ビルド**:
    ```poweshell
    cd my-app
    npm run build
    ```
2.  **静的ファイルの配置**:
    `my-app/out` フォルダの中身を、プロジェクトのルート (`yasuzuka-minami2.github.io/`) に全てコピー（上書き）する。
    ```powershell
    # ルートディレクトリで実行する場合の例
    xcopy "my-app\out" "." /E /Y
    ```
3.  **GitHubへプッシュ**:
    ```powershell
    git add .
    git commit -m "更新内容を記述"
    git push
    ```
