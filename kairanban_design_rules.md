# 回覧板詳細ページのデザイン・運用ルール

2026年2月14日制定

## 1. デザイン変更の指針
回覧板の内容を確認した後、確実に確認フォームへの入力を行ってもらうため、以下のルールを定める。

### 確認フォームの配置
- **配置場所**: ページ最上部（ヘッダー直下、PDF一覧の上）に配置する。
- **目的**: ユーザーがページを開いた直後にアクションを認識できるようにするため。

### 案内文言
- **文言**: 以下の文言に統一する。
  > 回覧板の内容をご確認いただきましたら、必ず下記のフォームからご連絡をお願いします。

## 2. ページ構成（並び順）
1. ヘッダー（タイトル・発行日）
2. **確認フォーム**（重要）
3. 全資料一括表示リンク
4. PDF資料一覧
5. 戻るボタン
6. フッター

## 3. デプロイ手順（Next.js）
デザイン変更後のデプロイは以下の手順で行う。

### 1. ビルド
`my-app` ディレクトリで以下のコマンドを実行する。
```bash
cd my-app
npm run build
```

### 2. 静的ファイルの配置
ビルド後、`out` ディレクトリ内のすべてのファイルをプロジェクトのルートディレクトリ（`yasuzuka-minami2.github.io` 直下）にコピーし、既存のファイルを上書きする。

**Powershellコマンド例**（`my-app` ディレクトリで実行）:
```powershell
Copy-Item -Path ".\out\*" -Destination "..\ " -Recurse -Force
```

### 3. 反映（Push）
変更をコミットして GitHub にプッシュする。
```bash
cd ..
git add .
git commit -m "Update site"
git push
```
